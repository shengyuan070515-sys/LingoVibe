|---|------|------|
| 1 | 四大类划分维度 | **按内容领域**（非场景 / 非考试标签专绑）。 |
| 2 | 类名与域名归属 | **全球时事**：theguardian.com, time.com, npr.org, vox.com, theatlantic.com；**科学与自然**：nature.com, newscientist.com, nationalgeographic.com；**科技前沿**：wired.com, technologyreview.com；**商业与财经**：economist.com, bloomberg.com, hbr.org。 |
| 3 | Aeon | **aeon.co → 全球时事**。 |
| 4 | 日界 | **`Asia/Shanghai`（北京时间）**；`dateKey` = 该时区 `YYYY-MM-DD`。 |
| 5 | 生成与缓存策略 | **混合**：Cron 每日定时预热为主；**当日无缓存时**读接口**兜底生成**；可接受换日后短空窗，产品可用骨架屏/准备中文案（细则实现阶段定）。 |
| 6 | 存储 | **KV / Redis 类**（如 Vercel KV、Upstash）；按 `dateKey` 存 JSON；写入需**幂等**（协调 Cron 与兜底）。 |
| **—** | **每日篇数（修订）** | **每个大类各 2 篇，共 8 篇**（原为各 1 篇共 4 篇）。**其余行为逻辑不变**（仍含决策 5、6 的 Cron/兜底/KV 等）。 |
| 7 | 精选「阅读原文」 | **① 先入库再读**：无则按 canonical URL **新建文章**再进阅读器；已存在则 **打开已有篇目**（与文库去重一致）。 |
| 8 | 正文抽取失败（付费墙等） | **策略 A + 明确退路**：仍 **入库并进入阅读器**；正文区展示 **已掌握的核心摘要**（优先使用精选/搜索阶段已有的摘要文本，如 Tavily `content`/snippet；实现阶段定义缺省时的降级文案）。摘要 **下方** 放置 **醒目按钮**，文案锁定为：**「原文有付费墙限制，已为您提取核心摘要。想看全文请点此去官网。」** 点击 **跳转原文 URL**（`canonicalUrl`，新标签打开 + `rel` 安全属性）。语气：**坦诚说明付费墙 + 突出已提供摘要价值**。 |
